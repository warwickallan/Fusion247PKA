## v3 CORRECTIONS (override v3 where they differ)
1. **"Direct reviewer access" = the trusted TOWER RUNTIME has direct read-only GitHub + Supabase-reviewer-view access — NOT the model subprocesses.** Codex/Fable receive the same *staged, resolved, canonical evidence snapshot + packet hash*. Neither model gets Supabase/DB/Telegram/cross-reviewer credentials. Keep the current adapter security: Fable tool-less in a neutral dir; Codex credential-stripped (Codex *may* have optional read-only repo access, but the staged packet is authoritative for both).
2. **`review_packet` persists the RESOLVED IMMUTABLE PAYLOAD** (or a content-addressed immutable artifact + hash) — NOT merely arrays of record IDs (a finding can change after packet creation; you must be able to reconstruct exactly what the model saw).
3. **No silent packet truncation.** If mandatory evidence can't fit/resolve → packet `BLOCKED`; require a PR split or an explicitly-approved bounded scope.
4. **Model-agnostic review ROLES.** Replace "Codex+Fable always required" with required ROLES: `product_qa`, `adversarial_assurance`, `security_assurance` (where applicable). Map authorised principals/models→roles via controlled config (Codex→product_qa, Fable→adversarial_assurance today; a future approved model must replace Fable without a schema rewrite). If an adversarial role is required and the reviewer is unavailable → **BLOCKED, never silently downgrade to Codex-only.**
5. **Telegram delivery** via a durable `notification_outbox` projection derived from `agent_event` (not a second news store): `id, source_event_id, notification_class, destination, dedup_key, headline, message, cockpit_url, github_url, state{queued|sending|sent|failed|dead_letter}, attempts, next_attempt_at, sent_at, last_error_code`. Classes: **ACTION_NEEDED** (Warwick decision / merge-live gate / Warwick-only blocker / material scope-privacy-cost-outcome change / autonomous work stopped pending authority); **MILESTONE** (meaningful WP built / final QA clean / real high-crit finding / merge landed / live activate-fail-recover-rollback / build complete-parked); **SILENT** (routine commits / intermediate reviews / ordinary fixes / CI progress / handoffs / repetitive ledger updates). Merge is always MILESTONE; Warwick-needed is always ACTION_NEEDED. One event → ≤1 notification per destination/class; bounded retry + dead-letter; no dup spam after restart; deep links to Directus + GitHub; notifier secrets outside reviewer processes; sent/failed/recovered events return to agent_event + cockpit.

## The final flow
Supabase+GitHub → trusted Tower packet-builder → immutable resolved snapshot + hash → same snapshot to Codex (+ Fable where required) → signed structured results → Tower validates identity/head/hash → Tower writes review_run/verdicts/findings/acceptance_verification → Supabase computes readiness → Directus displays → Telegram sends selected events.
