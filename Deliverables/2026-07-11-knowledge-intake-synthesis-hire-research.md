# Hire Research: Knowledge Intake and Synthesis Specialist

**Researcher:** Pax | **Briefed by:** Larry, per [[tsk-2026-07-10-003-categorisair-equivalent-design-proposal]] (Option A already approved by the user — this brief informs SOP-001 step 2, not whether to hire) | **Date:** 2026-07-11

## 1. What world-class looks like, day to day

This role sits at the intersection of three real disciplines: **taxonomy/controlled-vocabulary governance** (information architecture), **archival provenance discipline** (evidence-origin integrity), and **source-to-destination routing** (classic knowledge-management intake). None of it is hypothetical inside this build — CategorisAIr, `F247.update-fusion`, and `F247.template.source-classification-registry` are three real, if still-draft, prior attempts at exactly this job, and TubeAIR and the ICOR course-note template are two real, working capture channels already feeding it.

Day to day: an item arrives — an article, a PDF, a transcript, a pasted chat excerpt, a TubeAIR YouTube capture, an ICOR course note. The specialist (a) classifies it against a small, **governed** controlled vocabulary rather than inventing a category per item — professional taxonomy practice holds top-level categories to roughly 4–7 and grows them only from proven usage patterns, under an explicit owner/steward/contributor model so the vocabulary doesn't drift under casual, one-off editing ([Enterprise Knowledge](https://enterprise-knowledge.com/taxonomy-design-best-practices/); [taxonomystrategies.com](https://taxonomystrategies.com/governance/); [Glean](https://www.glean.com/perspectives/how-to-implement-effective-tagging-strategies-for-enterprise-data)); (b) picks the correct processing template and destination from that classification — the same job CategorisAIr's own registry and `F247.template.source-classification-registry` were built to do; (c) tags every extracted claim with where it actually came from — current context, a preserved raw source, or reconstructed from memory and therefore `Reconstructed / Needs verification` — exactly the discipline `F247.update-fusion`'s evidence rule encodes, and the same shape archival science calls "unbroken provenance": a record's evidentiary value depends on tracing it to its origin without a gap ([CLIR/Hirtle](https://www.clir.org/pubs/reports/pub92/hirtle/)); tiered evidence-confidence labeling of this kind is established practice, not an invented rule ([arXiv 2604.09296](https://arxiv.org/pdf/2604.09296)); (d) creates a backlink only when there's an articulable, knowledge-bearing reason — ungoverned, default-on tagging/linking is the most commonly cited cause of PKM rot ([Forte Labs](https://fortelabs.com/blog/a-complete-guide-to-tagging-for-personal-knowledge-management/); [dsebastien.net](https://www.dsebastien.net/ai-wiki-pkm-pkm-anti-patterns/)); and (e) files the result directly into its destination — never root, never a staging pile — the same "processed into knowledge, not hidden" principle already stated in the source system's own routing rules.

## 2. Core competencies and anti-patterns

- **Governed classification over per-item improvisation.** A small, stable vocabulary with an explicit owner beats inventing a new category every time something new arrives. Anti-pattern: a folder tree that grows a new one-off bucket per document — the exact failure the source system's own `Duplicate Folder Resolution` history shows happened repeatedly before cleanup.
- **Evidence-origin honesty over silent confidence.** Every extracted claim is labeled by where it actually came from; anything not traceable to current context or a preserved source is tagged `Reconstructed / Needs verification`, never written as settled fact. Anti-pattern: presenting a reconstructed memory with the same confidence as a directly-observed quote — the single failure mode `F247.update-fusion`'s hard rule exists to prevent.
- **Backlinks earn their existence.** A link is created because there is a stated, knowledge-bearing reason, not because the software makes linking free. Anti-pattern: reflexive, default-on tagging that produces a graph too dense to be navigable — the PKM community's most-cited anti-pattern for this exact discipline.
- **File it once, correctly, at the destination.** No root staging, no "I'll sort it later" pile. Anti-pattern, evidenced directly in this build's own source material: root-created source packs sitting unregistered until an explicit cleanup pass caught them.
- **One canonical intake process, many adapters.** TubeAIR and ICOR notes (and future channels — articles, PDFs, email, voice) own acquisition and handoff only; classification and synthesis logic live centrally, once, not duplicated per channel. Anti-pattern: a new "mini-specialist" or parallel pipeline spun up per content type — the opposite of what this hire is for.

## 3. Deliverables — adequate vs. world-class

Adequate: a source gets filed somewhere plausible, tagged with whatever category came to mind, linked wherever it seemed relevant. World-class: a source is classified against the governed vocabulary with the classification decision stated in one line; routed through the correct template on the first pass; every extracted claim carries an explicit evidence-origin label; every backlink carries a one-line justification a future reader could audit; and the artifact lands directly in its destination folder with no intermediate staging file. The differentiator is auditability — a skeptical reader should be able to ask "why does this link exist" or "how do we know this" and get a stated answer, not a shrug.

## 4. Boundaries (per the user's exact scope specification)

**Owns:** classification of already-acquired external sources; selecting the correct processing template and destination; source-to-WIKI synthesis; explicit evidence-origin labeling (including `Reconstructed / Needs verification`); justified backlink decisions; direct filing into the correct destination with no root staging; coordination of recurring intake adapters (TubeAIR, ICOR notes, and future channels — articles, PDFs, email, voice capture) as acquisition-and-handoff-only feeds into one canonical process.

**Refuses / hands back:** first-person personal-life capture → Penn. One-time bulk knowledge-base migrations → WS-002/Silas. Sources inside `Client Delivery/` → Warden. Independent research and truth verification (is a claim actually true, not just where it came from) → Pax. This specialist processes material already supplied to the Brain — it does not go find things in the world, and it does not manage client-delivery evidence.
